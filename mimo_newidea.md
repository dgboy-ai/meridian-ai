## Verified Enhancements (What to Build)

### Tier 1: High Impact, Verified Capabilities (Days 1-7)

#### 1. **Semantic Search for Playbook Retrieval**
- **DataHub Capability:** v1.6.0 Semantic Search with embedding providers
- **What:** Vector-based playbook retrieval on top of keyword search
- **Why:** More accurate pattern matching for reflexion loop
- **Implementation:** Use DataHub's semantic search API for playbook retrieval

#### 2. **Actions Framework Integration**
- **DataHub Capability:** Actions Framework (existing)
- **What:** Event-driven automation triggered by metadata changes
- **Why:** Real-time reactions to schema changes, freshness violations
- **Implementation:** YAML pipeline that triggers investigation on metadata events

#### 3. **ML Metadata Model Integration**
- **DataHub Capability:** MLModel, MLFeatureTable, MLFeature entities
- **What:** Use ML-specific entities for model health monitoring
- **Why:** Few hackathon participants know these exist — differentiator
- **Implementation:** Query MLModel entities for health scores, feature tables for drift

#### 4. **Structured Properties for AI Knowledge Panel**
- **DataHub Capability:** v1.6.0 Structured Properties with validation
- **What:** Typed custom metadata on model entities
- **Why:** AI Knowledge panel with validated fields
- **Implementation:** Write structured properties with health_score, confidence, playbook_reference

#### 5. **Assertions API for Quality Monitoring**
- **DataHub Capability:** v1.6.0 Assertions with failure severity
- **What:** Auto-create quality assertions for anomalies
- **Why:** Proactive quality monitoring, not just reactive detection
- **Implementation:** Create assertions when anomalies are detected

### Tier 2: Medium Impact, Novel Features (Days 8-14)

#### 6. **EU AI Act Compliance Engine**
- **What:** Automated audit trails for Article 12, 13, 14
- **Why:** Enforcement August 2, 2026 — we submit August 10
- **Implementation:** Cryptographic audit hashes, timestamped records, Technical File generation

#### 7. **PII/Compliance Scanner**
- **What:** Real regex-based scanner detecting email/SSN/IP/phone
- **Why:** Auto-raise compliance incidents with EU regulation citations
- **Implementation:** Already built in `scanners/pii_scanner.py` — integrate into Data Sentinel

#### 8. **Metadata-Aware dbt Model Generator**
- **What:** Generate dbt models using actual DataHub schemas
- **Why:** Opens Challenge 2 (Code Generation)
- **Implementation:** New worker that reads schema before generating

#### 9. **Shadow AI Discovery**
- **What:** Find models deployed without DataHub metadata
- **Why:** 82% of enterprises have unknown AI agents
- **Implementation:** Scan for MLModel entities with no structured properties

#### 10. **Contract Enforcer Worker**
- **What:** Auto-quarantine datasets violating contracts
- **Why:** Data contracts exist but nobody enforces them
- **Implementation:** Check assertions, propose quarantine lifecycle

### Tier 3: Differentiation (Days 15-21)

#### 11. **D3.js Blast Radius Visualization**
- **What:** Animated lineage graph with nodes lighting up sequentially
- **Why:** "This is the moment judges remember"
- **Implementation:** Canvas-based 60fps rendering

#### 12. **Knowledge History Panel**
- **What:** Shows pattern resolution time trend
- **Why:** Proves cumulative intelligence visually
- **Implementation:** Panel with incident history

#### 13. **Remediation Approval UI**
- **What:** APPROVE / OVERRIDE / REJECT for AI actions
- **Why:** EU AI Act Article 14 compliance
- **Implementation:** Modal with evidence chain

#### 14. **Executive Summary Export**
- **What:** One-click PDF/Markdown export
- **Why:** Serves every stakeholder
