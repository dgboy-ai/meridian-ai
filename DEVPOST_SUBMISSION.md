# Devpost Submission — Meridian AI

## Project Name
Meridian AI

## Tagline
AI Reliability Engineer that makes DataHub smarter every time an ML incident occurs.

## Challenge
Production ML Agents

## Description

Meridian AI is an AI Reliability Engineer that autonomously investigates production ML incidents, writes operational knowledge back into DataHub permanently, and ensures every future engineer — and every future AI agent — starts with everything the previous investigation learned.

### The Problem
- AI incidents rose 55% YoY (Stanford HAI 2026)
- 77% of ML teams have no monitoring tools
- Poor data quality costs $12.9M-$13.95M/year
- Context-aware platforms cut outage resolution by 58%

### The Solution
Every investigation becomes organizational memory. Meridian AI:
1. **Detects** — Schema changes, freshness violations, PII exposures, data quality issues
2. **Diagnoses** — Traverses DataHub lineage to find root cause, computes blast radius
3. **Remediates** — Proposes rollback, generates preventive assertions
4. **Learns** — Writes root cause reports, playbooks, AI Knowledge panels back to DataHub
5. **Complies** — EU AI Act SHA-256 audit chain for Articles 12, 13, 14

### DataHub Integration
- 24 capabilities used (15 read + 9 write)
- MCP Server for entity/lineage/schema access
- Write-back: AI Knowledge panel, root cause reports, playbooks, incidents
- Actions Framework YAML for auto-investigation on schema changes
- Reflexion playbook improves after every incident

### Technical Execution
- 21 workers, each computing real things (PSI, KS-test, schema diff, lineage traversal)
- Deterministic Validation Layer (4 checks before any write)
- Progressive Autonomy (5 levels from Advisory to Self-improving)
- Circuit breaker on LLM client
- Health score computed from real worker confidence
- 559 tests passing

### Originality
- **Cumulative intelligence** — Resolution time improves: 18min → 8min
- **EU AI Act compliance** — SHA-256 audit chain, Technical File generation
- **Self-healing assertions** — Generates preventive checks from incident patterns
- **Reflexion loop** — Self-RAG playbook improvement after every resolution

### Real-World Usefulness
- $45,000/day business impact (based on 32K predictions × $1.41/prediction)
- 4 personas: ML Platform Engineer, Data Engineer, MLOps Lead, Enterprise Data Architect
- EU AI Act enforcement August 2, 2026 (hackathon deadline August 11)

## How to Run

```bash
# Zero setup (mock mode)
git clone https://github.com/trueboy1123/meridian-ai
cd meridian-ai
pip install -e .
meridian investigate "urn:li:mlModel:(urn:li:dataPlatform:mlflow,churn_model_v3,PROD)"

# With real DataHub
docker compose up -d
python scripts/seed_meridian.py
python -m backend.main
```
